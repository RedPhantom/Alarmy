Introduction

Alarmy is designed to provide organization-wide reliable alarm capabilities. Alarmy services are offered by a central server.

This document contains operation instructions for the Alarmy Manager.

The application is still in its testing phase.

Future Capabilities

Following is a list of future Alarmy features.

  • Remote server management capabilities.
  • Settings management interface (instead of a text-based file).

Primary Interface

Information about the different parts and components in the application's primary interface.

Layout

The interface is divided into several parts, each part contains one or more sections.

From top to bottom, the first part - the header - contains the manager context menu.
The second part - from left to right - contains the Recipient List, Alarm Creation Panel and the Users and Groups Selection Panel.
The third part - the footer - contains the Server Status Indicator and the Hover Help Tooltip

Manager Context Menu

  • File
    • Start Server/Stop Server - control the server.
    • Settings - manage server settings and configuration.
    • Exit - stop and exit the application.
  • Deployment
    • Alarm Direction - control whether the alarm will be displayed to the user aligned to the right (RTL languages like Hebrew) or LTR.
    • Group Manager - manage the supported user groups on the server.
  • Help
    • Usage Guide - open this guide.
    • About - display version, author and diagnostic information.

Recipient List

List of connected users. Checked boxes affect which users will receive the deployed alarm. The list can be refreshed by clicking the Refresh button below it. User selection can be quickly made by the All (which selects all users) and None (which clears the recipient selections).

Note: user IDs are composed of the user's computer name and logged in user name. Multiple users can be logged in on the same computer (if it's a server, for example) and only specified user(s) on that server can be selected as recipients.

Note: the Recipient List doesn't rapidly refresh. A user may have disconnected from the server but still be displayed in the Recipient List and other parts of the application. Deploying an alarm to a user that has been disconnected will remove that user from the Recipient List. It's recommended to refresh the Recipient List before sending an alarm, if a specific user is targeted. A future feature may indicate which users should be connected to the server but have since lost connection.

Alarm Creation Panel

Composed of several fields, the Alarm Creation Panel is the interface in which the system administrator creates the alarms which will be deployed to recipient users.

The Alarm ID is a unique string which is used to differ one alarm from another. It is automatically generated and cannot be changed. The Alarm ID is used to coordinate between the system administrator and the user, as well as provide future alarm verification services.

The Title field contains the title of the alarm displayed to the user. It can only contain plain text.

The Content field contains the body of the alarm displayed to the user. It can contain plain or rich text. As the application doesn't contain a built-in rich text format editor, it's recommended that rich text alarms are first created in an application like Wordpad or Microsoft Word and then copied and pasted into the Content field.

Warning: Rich Text alarms allow for sending binary objects, including executable files. To protect users from malicious content, users cannot interact with Rich Text alarms. Limitations include deactivating hyperlinks and RTF object interaction. For organizations with security concerns or on systems connected to the internet, it's recommended to use Text Only alarms.

The Send button deploys the alarm across the selected recipients in the Recipient List.

Users and Groups Selection Panel

This panel contains two tabs: Users and Groups.

The Users tab lists all users that have logged on to the server and the time of last contact. Users with an especially old last contact time are marked.

The Groups tab lists all groups that support alarm deployment. There needs to be at least one connected user of that group for that group to appear in the Groups tab.

The Target Selected button will apply the selected Users or Groups (depends on the selected tab) to the Recipient List.

Server Status Indicator

This indicator shows the server status: Starting Up, Running or Stopped.

Note: If the server appears to be "stuck" in the Starting Up phase, check the Server Information Console. The server may be waiting for you to type the certificate password.

Server Information Console

This console opens alongside the primary interface. It is used to report and display system events to the system administrator. The severity of events displayed can be configured in the Settings menu available under the File menu in the Primary Interface.

On sysytems with a certificate configured, the Server Information Console will wait for the sysytem administrator to type the certificate password before the startup procedure can continue. Until then, the server status will be displayed as Starting Up.

Group Manager

The Group Manager allows the system administrator to add, edit and remove groups to which users can associate themselved.

To add a group, type a group name in the designated textbox and click Add or TAB.
To remove a group, select a group and click Remove.
To edit a group, click it, type its new name in the designated textbox name and click Edit.

Save your changes by clicking Apply.

Note: editing a group's name doesn't change it's ID. The modified name will only be shown to new clients, or clients that refresh their group list.
